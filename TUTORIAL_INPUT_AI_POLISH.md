# Crown's Bane — Туторіал: Виправлення інпуту, AI, візуальний поліш

> **Передумови:** ти пройшов попередні туторіали. Корабль є в рівні як `BP_PlayerShip`.
> **Час:** ~30 хв.

---

## ЧАСТИНА 1 — ВИПРАВЛЕННЯ ІНПУТУ (Найважливіше)

Якщо клавіатура / миша не реагують — дві причини:

### Причина A: GameMode спавнив "порожній" корабль

Раніше `DefaultPawnClass = AShipPawn::StaticClass()`. UE знаходив PlayerStart і
спавнив голий C++ AShipPawn без жодних асетів (без IMC_Ship, без IA_*), і
гравець керував ним. Зараз виправлено:

```
DefaultPawnClass = nullptr
```

Тепер GameMode **нічого не спавнить** — BP_PlayerShip сам себе посесить через
`AutoPossessPlayer = Player0`, встановлений у C++ конструкторі.

**Що тобі треба:** переконатись, що у рівні є **саме один** BP_PlayerShip і
немає PlayerStart з прив'язаним AShipPawn.

### Причина B: PIE не ловив кліки мишки

PlayerController не переключав режим інпуту на Game Only. Виправлено в
`ACrownsBanePlayerController::BeginPlay()`:

```cpp
bShowMouseCursor = false;
SetInputMode(FInputModeGameOnly());
```

Курсор тепер ховається одразу при старті сесії. Всі кліки та рухи миші йдуть
безпосередньо в гру — **не потрібно клікати на viewport**.

### Причина C (якщо досі не працює): IMC_Ship не призначений

Якщо після цих фіксів все ще мовчить — перевір:

1. Відкрий `BP_PlayerShip → Class Defaults → Input`
2. Поле `Ship Mapping Context` повинно бути = `IMC_Ship`
3. `IA_IncreaseSail`, `IA_DecreaseSail`, `IA_Turn`, `IA_FireLeft`, `IA_FireRight`
   повинні вказувати на відповідні IA-асети

Якщо будь-яке поле NULL — відповідна кнопка мовчатиме, а в Output Log з'явиться
повідомлення `[ShipPawn] IA_IncreaseSail unassigned!` (і т.д.).

**Debug:** запусти гру, дивись на зелене повідомлення `IMC registered: IMC_Ship`
у лівому куті екрана. Якщо бачиш червоне — читай конкретну причину.

## ЧАСТИНА 2 — ЕФЕКТИ ПРИ ПОПАДАННІ ТА СТРІЛЬБІ

### 2.1. Camera Shake при попаданні

**BP_PlayerShip → Class Defaults → FX:**
- `Hit Camera Shake` = будь-який `UCameraShakeBase` Blueprint
- `Hit Camera Shake Scale` = 1.5 (можна підкрутити)

Якщо немає готового Blueprint Shake — створи:
**Content Browser → Blueprint Class → All Classes → пошук `MatineeCameraShake`
або `LegacyCameraShake`**. Назви `BS_HitShake`. Відкрий → встанови:
- `Oscillation Duration` = 0.3
- `Rot Oscillation → Pitch → Amplitude` = 3, `Frequency` = 20

Постав цей BS_HitShake у поле `Hit Camera Shake` BP_PlayerShip. Тепер кожне
попадання ядром буде трясти камеру пропорційно до розміру пошкодження.

### 2.2. Розкид гармат

Кожне ядро тепер відхиляється на ±2° по горизонталі і ±0.6° по вертикалі
від ідеального напряму. Налаштовується в:
**BP_PlayerShip (або BP_EnemyShip) → CannonComponent → Cannon Spread Angle** (за замовч. = 4°).

Це робить залпи природнішими та тактично цікавішими: ядра більше не летять
точно в одну точку.

## ЧАСТИНА 3 — ЕФЕКТИ УШКОДЖЕНЬ ВОРОГІВ

Ворожі кораблі тепер отримують такі ж FX як і гравець.

### 3.1. Відкрий кожен BP ворога (BP_Sloop, BP_Brig, BP_Galleon)

**Class Defaults → FX** → заповни ті самі поля, що і в BP_PlayerShip:

| Поле              | Що ставити                        |
|-------------------|-----------------------------------|
| `Smoke Asset`     | Niagara димового шлейфу           |
| `Fire Asset`      | Niagara вогню                     |
| `Death Explosion Asset` | Великий Niagara вибуху      |
| `Death Sound`     | Звук вибуху/загибелі              |
| `Smoke HP Threshold` | 0.6 (60% HP → дим)             |
| `Fire HP Threshold`  | 0.3 (30% HP → вогонь)          |

Якщо не заповнювати — FX просто не грають, ігровий процес не ламається.

## ЧАСТИНА 4 — НОВИЙ ІІ: СТАН RETREAT (ВІДСТУП)

### 4.1. Як це виглядає

Кожен ворожий корабль тепер має 5 станів:
`Patrol → Chase → Attack → **Retreat** → Sink`

Коли HP падає нижче `RetreatHealthThreshold` (за замовч. 25%), корабль:
1. Перестає стріляти
2. Повертається спиною до гравця
3. Тікає в бік свого спавну з підвищеною швидкістю (`RetreatSpeed = 700`)
4. Якщо відірветься від гравця (> 2× DetectionRange) — повертається у Patrol

### 4.2. Налаштування в BP

| BP          | `Retreat Health Threshold` | `Retreat Speed` | `bCanRetreat` |
|-------------|---------------------------|-----------------|---------------|
| BP_Sloop    | 0.30 (рекомендую)         | 900             | true          |
| BP_Brig     | 0.25                      | 700             | true          |
| BP_Galleon  | **ігнорується**           | —               | false†        |

†Галеон **ніколи не відступає**. Коли HP падає до `EnrageHealthThreshold` (30%)
він навпаки входить у режим Enrage — прискорюється і починає стріляти частіше.
Коли HP нижче 25% і він ще не в Enrage — спочатку перевіряє Enrage, потім
вирішує. Якщо вже enraged — HandleStateRetreat викликає HandleStateAttack.

### 4.3. Рекомендований патерн бою

- **Sloop**: щупальце-і-відступ. Кілька залпів зблизька, потім тікає → пройдеш
  за ним і добереш. Тактика: впіймати в лаг (тоді повільніший).
- **Brig**: агресивно закривається, відступає тільки при 25%. Дай йому бортовий
  залп на зближенні — потрапиш у лоб.
- **Galleon**: бос. Ніколи не здасться. Enrage при 30% → стає швидшим, рідше
  перезаряджає. Кращий час для золотого пострілу — поки не enraged.

## ЧАСТИНА 5 — ПОЛОСЫ HP НАД ВОРОГАМИ (HUD)

Повністю автоматично — нічого не треба налаштовувати.

**Що видно** (якщо ворог ближче ніж 80 м):
- Горизонтальна HP-смужка (зелена → червона) над кораблем
- Назва актора + відсоток HP
- Поточний стан AI у дужках: `[PATROL]`, `[CHASING]`, `[ATTACK]`, `[FLEEING]`, `[SINKING]`

**Налаштувати дальність і розмір:**
Відкрий `BP_CrownsBaneHUD → Class Defaults → HUD|Layout`:
- `Enemy HP Draw Range` = 8000 (cm = 80 м за замовч.)
- `Enemy HP Bar Width` = 100
- `Enemy HP Bar Height` = 8

## ЧАСТИНА 6 — УСУНЕННЯ ТИПОВИХ ПРОБЛЕМ

### Інпут все ще мовчить

1. Відкрий вікно Output Log (Window → Developer Tools → Output Log)
2. Шукай рядки `[ShipPawn]`
3. Якщо бачиш `Controller=NONE` → BP_PlayerShip не посесено → перевір, чи в рівні
   є BP_PlayerShip (не просто C++ AShipPawn)
4. Якщо бачиш `IMC registered: IMC_Ship` (зелений текст на екрані) але все одно
   мовчить → можливо, одна з IA_* властивостей NULL

### Корабель спавниться але стоїть на місці

Перевір логи на `[ShipPawn] IA_Turn unassigned!` або подібні. Кожна незаповнена
IA-властивість у BP_PlayerShip → та клавіша не реагує.

### Ворожі кораблі не відступають

- Перевір `bCanRetreat = true` в **BP_Sloop / BP_Brig** (не в Galleon!)
- Переконайся, що `RetreatHealthThreshold > 0.0`
- Якщо грають на піратські кораблі — вони можуть не доживати до 25%

### Трясіння при попаданні не відчувається

- `Hit Camera Shake` у BP_PlayerShip повинно вказувати на Shake-Blueprint
- Переконайся що у Shake-Blueprint є ненульові значення амплітуди

## Готово!

Після цього туторіалу:
- ✅ Інпут (клавіатура + миша) повинен **нарешті** стабільно працювати в PIE
- ✅ Гармати мають природній розкид, що робить перестрілки менш "роботоподібними"
- ✅ Ворожі кораблі димляться/горять/вибухають при загибелі
- ✅ AI ворогів знає коли тікати, а Galleon бореться до кінця
- ✅ Навів HUD над кожним ворогом — бачиш стан і HP без здогадок
- ✅ Потрапляння в твій корабель трусить камеру пропорційно до урону

Наступні очевидні кроки:
- Додати музику / ambient (вітер, хвилі, морські птахи)
- Зробити **мінімапу** з іконками ворогів і скарбів
- Додати **туман/видимість** під час шторму через post-process
