## ЧАСТИНА 1 — Мінімапа

### Як це працює

HUD малює круглу мінімапу в нижньому лівому куті (`MinimapCenter`, `MinimapRadius = 120`).
Вона завжди "вгору = ніс корабля" — карта обертається відносно корабля.

```
Rotation: -ShipYaw (щоб корабель завжди дивився вгору)
```

На мінімапі видно:
- **Жовтий трикутник** — ваш корабель
- **Конус огляду камери** (прозорий) — куди дивиться камера
- **Червоні точки** — вороги
- **Зелені точки** — скрині/лут
- **Сині точки** — доки

### Дуги вогню гармат

По боках корабля HUD малює **жовту дугу** — зону, куди можна стріляти.
Якщо камера дивиться в бік → потрібна сторона підсвічується яскравіше.

```cpp
// У CrownsBaneHUD.cpp
void ACrownsBaneHUD::DrawFiringArcs() { ... }
```

### Що треба в Blueprint

Нічого. Мінімапа працює автоматично. Можна налаштувати в деталях HUD:

| Властивість | Дефолт | Опис |
|---|---|---|
| `MinimapRadius` | 120 | Розмір кола |
| `MinimapWorldRange` | 8000 | Скільки юнітів "влазить" в карту |
| `MinimapCenter` | (140, 140) | Позиція на екрані |

## ЧАСТИНА 2 — Прицілювання за камерою

### Стара система
`Q` — ліва сторона, `E` — права сторона. Незручно.

### Нова система
**ЛКМ / Пробіл** — стріляє в сторону, куди дивиться камера.

```cpp
void AShipPawn::Input_Fire(const FInputActionValue& Value)
{
    const float DotRight = FVector::DotProduct(
        Camera->GetForwardVector(), GetActorRightVector());

    if (DotRight > 0.35f)       CannonComponent->FireBroadside(ECannonSide::Right);
    else if (DotRight < -0.35f) CannonComponent->FireBroadside(ECannonSide::Left);
    // якщо камера дивиться вперед/назад — нічого не відбувається
}
```

Порогове значення `0.35` ≈ 20° від бортової лінії. Тобто камера повинна
дивитися "збоку" від корабля щоб постріл зарахувався.

**Q/E все ще працюють** для ручного вибору сторони.

## ЧАСТИНА 3 — Агресія ворогів (Wanted Level)

### Поведінка

| Wanted Level | Поведінка ворогів |
|---|---|
| 0 | Патрулюють, **ігнорують** гравця |
| 1-2 | Переслідують, якщо гравець близько |
| 3+ | Активно атакують |

### bHasAggro

Коли ворожий корабель **отримує урон від гравця**, він встановлює `bHasAggro = true`.
Після цього **цей конкретний корабль** атакує незалежно від Wanted Level.

```cpp
// EnemyShipBase.cpp — TakeDamage
if (Cast<AShipPawn>(DamageCauser))
{
    bHasAggro = true; // цей ворог тепер агресивний
}
```

Тобто: можна випадково потрапити по ворогові → він почне атакувати, але інші
кораблі в районі залишаться мирними (якщо Wanted == 0).

## ЧАСТИНА 4 — Фолбек інпуту (завжди працює)

### Проблема
Якщо в `BP_PlayerShip` не призначені `ShipMappingContext`, `IA_Turn` і т.д. —
гра не реагувала на клавіатуру.

### Рішення: `EnsureInputAssetsExist()`

Викликається в `BeginPlay`, `PawnClientRestart`, `NotifyControllerChanged` і
`SetupPlayerInputComponent`. Якщо BP-поля порожні — створює всі асети програмно:

```
W       → IncreaseSail
S       → DecreaseSail
A/←     → TurnLeft  (з модифікатором Negate)
D/→     → TurnRight
Q       → FireLeft
E       → FireRight
ЛКМ / Пробіл → Fire (camera-aim)
```

**Результат:** гра повністю грається без жодної конфігурації в Blueprint.

## ЧАСТИНА 5 — Зміна дня і ночі

### Системa

`ADayNightSystem` (новий актор) шукає `ADirectionalLight` у рівні через
`TActorIterator` і обертає його кожен тік:

```
Pitch сонця: -90° (північ) → +90° (полудень) → -90° (наступна ніч)
```

Цикл займає `DayLengthSeconds = 600` секунд (10 хвилин реального часу).

### Фази

| Час | Фаза | Колір неба | Інтенсивність |
|---|---|---|---|
| 0.0 — 0.25 | Ніч | Темно-синій | 0.1 |
| 0.25 — 0.35 | Світанок | Помаранчевий | 0→3 |
| 0.35 — 0.65 | День | Білий | 3.0 |
| 0.65 — 0.75 | Захід | Помаранчевий | 3→0.1 |
| 0.75 — 1.0 | Ніч | Темно-синій | 0.1 |

### Що треба в рівні

Обов'язково: **один Directional Light** у рівні (він там є за замовчуванням в
Ocean/SunSky template). GameMode спавнить DayNightSystem автоматично.

Можна налаштувати:
```cpp
// CrownsBaneGameMode.h
TSubclassOf<ADayNightSystem> DayNightSystemClass; // перевизначити в BP
```

## ЧАСТИНА 6 — Система боєприпасів

### Інвентар

```cpp
// PlayerInventory.h
int32 CannonAmmo = 40;   // поточний заряд
int32 MaxCannonAmmo = 40; // максимум
```

**Методи:**
- `AddAmmo(int32 Amount)` — додати (з кліпом по максимуму)
- `ConsumeAmmo(int32 Amount) → bool` — витратити (повертає false якщо нема)
- `UpgradeMaxAmmo(int32 Bonus)` — апгрейд максимуму

### Спалення патронів

`CannonComponent` перевіряє ammo перед кожним залпом:

```cpp
// CannonComponent.cpp — FireBroadside
UPlayerInventory* Inv = /* lookup */;
if (Inv && !Inv->ConsumeAmmo(CannonsOnSide))
{
    // нема патронів — тихо виходимо
    return;
}
```

Тобто кожна гармата = 1 патрон. Якщо є 4 гармати → потрібно 4 патрони.

### Лут патронів

Кораблі при смерті дропають ammo pickups:

| Тип ворога | Ammo drop |
|---|---|
| Sloop | 6 |
| Brig | 12 |
| Galleon | 24 |

### HUD

Лічильник показується в правому нижньому куті: `40 / 40 ⦿`
Якщо < 25% — змінюється на червоний.

### Покупка в доку

Через існуючий `UDockInteraction` можна купити патрони так само як інші
ресурси (Gold → Ammo). Реалізація через `AddAmmo()`.

## ЧАСТИНА 7 — Анімація потоплення

### 3 фази (6 секунд)

```
Фаза 1 (0→2s): корабль повільно нахиляється → крен +25°
Фаза 2 (2→4s): ніс занурюється → pitch -45°, крен +35°
Фаза 3 (4→6s): корма злітає → pitch -70°, і корабль іде під воду
```

Використовує `FMath::InterpEaseOut` / `FMath::InterpEaseIn` для плавності.
Після 6 секунд корабль знищується (`Destroy()`).

### FX під час потоплення

- Smoke і Fire FX залишаються активними під час анімації
- Death Explosion спрацьовує на початку потоплення

### HullArmor — броня корпусу
| Tier | Золото | Дерево | Метал | Ефект |
|---|---|---|---|---|
| 1 | 180 | 15 | 15 | -5% dmg |
| 2 | 400 | 35 | 35 | -10% dmg |
| 3 | 750 | 60 | 65 | -15% dmg |
| 4 | 1300 | 100 | 120 | -25% dmg |

Броня працює **мультиплікативно** та стекується:
```cpp

{
    const float MitigatedDamage = DamageAmount * (1.0f - ArmorReduction);
    return Super::TakeDamage(MitigatedDamage, ...);
}
```

## Відомі обмеження

- `DayNightSystem` шукає перший `ADirectionalLight` у рівні; якщо їх декілька —
  можуть бути конфлікти. Рекомендується один основний Directional Light.
- Мінімапа не показує острови/суходіл (лише динамічні актори).
- Броня не захищає від падіння або загибелі поза боєм.
